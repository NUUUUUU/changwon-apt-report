## 2. 데이터 소스 — 검증된 접근 방법

### 2-1. 네이버부동산 (주력, 실측 성공 ✅)
- **지역코드(cortarNo)**: `GET https://m.land.naver.com/map/getRegionList?cortarNo={상위코드}`
  - 경남=`4800000000` → **창원 성산구=`4812300000`** → 동별 코드 확보(예: 가음정동 `4812310300`, 남산동 `4812310600`).
- **토큰 추출**: `GET https://new.land.naver.com/` 응답 HTML에서 JWT(`eyJ…` 정규식) 추출.
- **매물 조회**: `GET https://new.land.naver.com/api/articles?cortarNo={코드}&order=rank&realEstateType=APT&tradeType=A1:B1&priceMax=30000&page={N}`
  - 헤더: `Authorization: Bearer {JWT}`, `Referer: https://new.land.naver.com/`
  - 응답 필드: `articleName`(단지), `tradeTypeName`+`dealOrWarrantPrc`(가격), `area2`(전용㎡), `floorInfo`(층), `buildingName`(동), `articleNo`(→ 매물링크).
- **TLS 위장**: `curl_cffi`(`impersonate="chrome131"`) — insane-search가 설치한 라이브러리.
- ⚠️ 위험: 네이버가 토큰/Referer 정책 바꾸면 수정 필요(아래 §9).

### 2-2. 당근부동산 (보조, 가능·파서 필요 ⚠️)
- 매물 데이터가 페이지 HTML에 임베드: `window.RELAY_STORE` + `application/ld+json`(JSON-LD). **로그인 불필요**.
- 흐름: 지역명→당근 내부 region id 해석 → `GET https://www.daangn.com/kr/realty/?in={지역명}-{id}` → HTML에서 RELAY_STORE/JSON-LD 파싱.
- 특성: 직거래·빌라/투룸 위주 → **아파트 매매 인벤토리는 네이버보다 적음**. 보강용.
- 참고 오픈소스: NomaDamas/k-skill `daangn-realty-search`.

### 2-3. 국토부 실거래가 (가격 신뢰선, 공식 ✅)
- `GET http://apis.data.go.kr/1613000/RTMSDataSvcAptTrade/getRTMSDataSvcAptTrade?serviceKey={KEY}&LAWD_CD=48123&DEAL_YMD={YYYYMM}` (아파트 매매)
- 전월세: `…/RTMSDataSvcAptRent/getRTMSDataSvcAptRent`
- `LAWD_CD`=법정동 5자리(**창원 성산구=48123**, 의창구 48121, 마산회원 48127, 마산합포 48125, 진해 48129).
- ⚠️ **요청 시 `User-Agent` 헤더 필수** — 없으면 WAF가 `Request Blocked`로 차단(실측 확인). 응답 XML, 금액 단위 **만원**.
- 용도: 매물 호가가 **최근 실거래 대비 저평가/고평가**인지 판단.

### 2-4. 한국부동산원 REB (중위/평균가, 공식 ✅)
- OpenAPI: 시군구 중위/평균가(`STATBL_ID` 예: 매매 중위가 `A_2024_00189`, 평균가 `A_2024_00188`).
- 용도: 단지 호가를 지역 중위가와 비교 → 저평가 점수 보정.

### 2-5. 보강 데이터
| 항목 | 소스 | 엔드포인트(표준) |
|---|---|---|
| 좌표(지오코딩) | 카카오 로컬 | `https://dapi.kakao.com/v2/local/search/address.json` |
| 통근 시간 | 카카오 모빌리티 | `https://apis-navi.kakaomobility.com/v1/directions` (origin=매물, dest=공장 128.6962,35.1981) |
| 학군 | NEIS | `https://open.neis.go.kr/hub/schoolInfo` + 학교좌표(카카오 로컬) |
| 상권 POI | 카카오 로컬 카테고리 / 네이버 지역검색 | `…/local/search/category.json` (MT1 마트, HP8 병원, …) |

> 표준 REST API(키 발급형)를 쓰는 이유: **standalone 스크립트는 MCP에 의존 못 함**. MCP는 프로토타이핑/대화용, 자동 실행은 직접 API.

## 5. 실행 순서 (main.py 오케스트레이션)

1. `config.yaml`·`.env` 로드
2. **수집**: 네이버 + 당근에서 대상 동코드들의 매물(≤3억, 매매/전세) → `List[Listing]`
3. **중복 제거**: (단지+동+층+전용+가격) 키로 dedupe, source 우선순위 네이버>당근 *(R2-3)*
4. **재검증(R2)**: 존재 재확인·주소/좌표 정합·이상치 → 제외 또는 ⚠️플래그
5. **가격 기준선**: 국토부 실거래 + REB 중위가 → 단지/면적별 `fair_price` 산출
6. **보강**: 각 매물 → 지오코딩 → 통근분 → 학군/육아 → 상권 POI
7. **하드필터(§A2)**: 통근≤30분·예산·면적·연식·육아 최소요건 → 통과셋
8. **점수화**: §6 공식 → 종합점수
9. **다양성(R1)**: 단지당 대표 1매물 + 최근 N일 쿨다운 → Top 10 랭킹
10. **차선(R3)**: 통과<10이면 완화 사다리로 부족분 채움(별도 "🔶차선" 섹션)
11. **리포트**: `docs/{날짜}.html` 생성 + `docs/index.html` 갱신 + 스냅샷 저장
12. **발행**: `git add docs && commit && push` → GitHub Pages
13. **발송**: 카카오 나에게 보내기(요약 + 링크)
14. **로그**: 수집·검증·통과·차선·발송 결과 기록

## 5.5 운영 규정 (Operating Rules)

### R1. 단지 다양성 — "같은 단지 연속 추천 금지" *(확정)*
- **리포트 내**: 1개 단지 = **대표 1매물(최고점)** 만 Top 10 진입 → 10개 모두 **다른 단지** 보장.
- **날짜 간 쿨다운**: 최근 **3일** 추천된 단지는 **제외/강등**. 예외 = ①가격 인하 ②더 나은 매물 재등장 ③후보 풀 고갈. (일일 스냅샷 히스토리 활용)
- 목적: 매일 **새로운 단지를 탐색**.

### R2. 매물 재검증 — "조건·실재 여부 재확인"
점수화 전 각 후보 자동 검증 → 실패 제외 / 의심 플래그:
1. **존재 재확인**: 매물 상세 재호출(네이버 `article/info`) → 활성·거래완료 아님. 404/거래완료/삭제 → **제외**.
2. **주소·좌표 정합**: 카카오 지오코딩 → 단지명↔주소↔좌표 일치 & 통근권(성산구 등) 내 확인.
3. **중복 제거**: 다중 중개 중복 1건화.
4. **허위/미끼 신호**: 호가가 실거래 대비 비정상(예: −40%↓), 장기 미갱신, 동일조건 과다 → ⚠️"검증주의"(제외 아님, 경고).
- 한계: 자동 검증(존재·주소·중복·이상치)까지. 전화/현장 확인은 사용자 몫 → 리포트에 **"현장확인 권장"** 명시.

### R3. 차선(근접) 매물 — "충족 매물 없을 때"
- 하드필터 통과 < 10건이면 **완화 사다리**로 부족분 채움. 충족분과 **별도 섹션("🔶 차선")** + 각 매물에 **미달 사유 배지**(예: "통근 34분·기준+4").
- **완화 우선순위 — 보수적(확정)**: ① 통근 30→**35분** → ② 면적 전용 **72~74㎡까지 소폭**(그 이하 불가)
- **절대 유지(완화 안 함)**: **예산 3억** · **디딤돌 면적(전용 85㎡)** · **연식**(35년 초과 제외, 25년+ 비수리는 저평가 '상'만). → 돈·자격 관련은 절대 안 건드림.
- 0건이어도 "조건 없음" 대신 **가장 근접한 후보**(미달 사유 명시)를 제시.

## 6. 점수화 방법 (score.py)

각 항목 0~100 정규화 후 가중합 (가중치 `config.yaml`에서 조정). **하드필터(§A2) 통과 매물만** 점수화.

| 축 | 가중치 | 산식(요지) |
|---|---|---|
| **가격/저평가** | 30 | `(fair_price - 호가)/fair_price` 클수록↑ (국토부·REB 대비 싼 매물 우대) |
| **교통/통근** | 25 | 공장까지 차량분: 10분↓=100, 30분=0 선형. 30분 초과는 **하드 제외** |
| **육아 인프라** | 25 | **어린이집·유치원 + 초등학교(배정·도보거리)** 최우선 가중, 소아과·공원 보조 (거리·개수) |
| **상권/생활편의** | 20 | 반경 1km 내 마트·병원·카페 POI 수 정규화 |

> 동점 시 저평가 > 통근 순. 4·6세 자녀 양육이 핵심이라 **육아 인프라를 학군보다 상위 축으로 승격**.

## 7. 리포트(HTML) — 매수자 관점 구성

스타일: 맑은 고딕, 네이비(1F3864), **모바일 우선**(카톡 링크로 폰에서 열람), 카드형.

> **확정(2026-06-27)**: 레이아웃 = **정보 풍부형 카드** · 지도 = **카카오맵 임베드** · 추가기능 = **변동 하이라이트(신규·인하) + 디딤돌 대출 추정 + 전세가율·갭 + 단지 6개월 시세 추이** (4개 모두 채택).

### 7-1. 매수자가 필요한 정보 = 4개 질문에 답하는 구조
"① 싼가? ② 살 수 있나(대출)? ③ 출퇴근 되나? ④ 애 키우기 좋나?"

**① 상단 요약 대시보드**
- 날짜 + 제목 (상단 필터 조건 칩은 **제거** — 조건 고정이라 생략)
- 오늘의 한 줄: 신규 N · 저평가'상' M · 통근15분내 K

**보조 메모**
- **가격 매력도 등급**: 저평가율 = (국토부/REB 기준가 − 호가) ÷ 기준가. **상 ≥10% · 중 3~10% · 하 <3%** (`config.yaml` 조정). → 25년+ 비수리 구축은 **'상'만 채택**, 그 외 매물은 가격축 점수에 반영.
- 육아 인프라 가중: **어린이집/유치원 + 초등학교(배정·도보거리)** 최우선, 소아과·공원은 보조 가점.
